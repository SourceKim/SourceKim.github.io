iOS 中 weak 详细介绍

 Posted on May 8, 2019  |  1 minutes  |  75 words  |  Kim

Runtime 为我们维护了一张全局的 weak 表（弱引用表），是一张 哈希表，该表的 key 是要弱引用的对象的地址， value 是所有指向该对象的 weak 指针，是一个数组。

比如：

    id __weak ptr = obj;

那么存入的的 key 是 obj 的地址，要添加到 value 数组的值是 ptr 的地址。

一、Runtime 的调用顺序

    1. 先调用 objc_initWeak
    1. 再调用 objc_storeWeak

    • weak_register_noLock

      • 通过 hash 运算生成 weak_entry_t 结构体，然后存入 weak 表中

      • 最后将该对象的 weakReference 属性置为 true

    • weak_unregister_noLock

二、weak 属性是怎么被销毁的？

  1. 如果对象有被 weak 指针指向的话（weak 修饰的对象），在对象销毁释放内存执行 -dealloc 方法的时候，

  2. 根据函数的调用顺序会执行到 weak_clear_no_lock 这个方法

  3. 获取到对应对象的 weak_entry_t *entry

  4. 根据 entry 在 weak 表 找到 value，这个 value 保存着所有 weak 指针的地址

  5. 遍历 value，将所有指针置为 nil

  6. 最后清理掉这个 entry
