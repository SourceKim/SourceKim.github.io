Block 原理 & 本质

 Posted on January 17, 2019  |  2 minutes  |  302 words  |  Kim

iOS 的 block 本质上也是一个 OC 的对象，是一个拥有 isa 指针的结构体，能够捕获外部变量的值、指针地址。

一、内存中表现形式

在内存中是一个 结构体： __main_block_impl_0

struct __main_block_impl_0 {
  struct __block_impl impl;
  struct __main_block_desc_0* Desc;
  __main_block_impl_0(void *fp, struct __main_block_desc_0 *desc, int flags=0) {
    impl.isa = &_NSConcreteStackBlock;
    impl.Flags = flags;
    impl.FuncPtr = fp;
    Desc = desc;
  }
};

包含两个结构体成员：__block_impl & __main_block_desc_0

  • __block_impl block 的主要实现

    struct __block_impl {
void *isa;
int Flags;
int Reserved;
void *FuncPtr;
};

    包含 isa，还有 FuncPtr 指向 block 所封装的代码块地址，等执行 block 时会通过 FuncPtr 寻找将要执行的代码块，并且调用

    • __main_block_desc_0 描述结构
    static struct __main_block_desc_0 {
size_t reserved;
size_t Block_size;
}

    Block_size 为当前block 占用内存大小

block 中的代码块是 __main_block_func_0 结构体

二、变量捕获

  1. auto(普通)变量

    直接存储该变量的值

  2. static 变量

    存储该变量的指针

  3. 全局变量

    不捕获 （直接访问全局区域）

三、block 类型

block 类型分三种：分别为

  • __NSGlobalBlock__ 全局 block
  • __NSMallocBlock__ 堆 block
  • __NSStackBlock__ 栈 block

这三种 block 都继承自 NSBlock，而 NSBlock 继承自 NSObject

定义 block 的时候会使用哪个结构，被下面的条件决定

类型条件
NSGlobalBlock没有访问（捕获）普通变量
NSMallocBlockNSStackBlock 执行了 copy
NSStackBlock访问（捕获）普通变量

当 ARC 下，下列情况会自动对 block 执行 copy

    1. 当 block 作为函数返回值时
    1. 当 block 被强指针引用时
    1. 当 block 做为 cocoa API 或 GCD API 的方法参数时

对不同的 block 调用 copy：

  • __NSGlobalBlock__ 调用copy操作后，什么也不做
  • __NSStackBlock __ 调用copy操作后，复制效果是：从栈复制到堆；副本存储位置是堆
  • __NSStackBlock__ 调用copy操作后，复制效果是：引用计数增加；副本存储位置是堆

四、修改外部变量

  1. 使用 static

  2. 使用 __block 修饰

    使用 __block 修饰的对象，在声明的时候会被转化为 __Block_byref_a_0 对象

    其结构为：

    struct __Block_byref_a_0 {
    void *__isa;
    __Block_byref_a_0 *__forwarding;
    int __flags;
    int __size;
    int val;
};

    其中 __forwarding 为原变量的地址，val 为原来变量的值

    然后把这个值传入到 block 中，在 block 内对该值作更新的时候，实际上是修改 __forwarding 指向的地址里面的值，从而修改外部变量的值。

五、访问 auto 变量 （堆 & 栈 block 的区别）

  1. 对于 栈 的 block，不会对 auto 变量进行强引用

  2. 如果 block 被 copy 到 堆 上：

    • a、 先调用 block 的 copy

    • b、 copy 调用 _block_object_assign，然后根据 auto 的修饰符（retain、weak等），形成强引用或者弱引用

  3. 如果 block 从 堆 上移除：

    • a、先调用 block 的 dispose

    • b、 dispose 调用 _block_object_dispose，去释放引用的变量

六、循环引用

如果 block 引用了一个对象，这个对象又引用了 block，就会造成循环引用，可以通过如下解决：

  • ARC：

      1. 使用 __weak，_unsafe_unretain 去修饰这个对象
      1. 使用 __block 去修饰这个对象，然后在 block 内部将这个对象置为 nil （因此一定要调用这个 block）

  • MRC:

      1. 使用 _unsafe_unretain 去修饰这个对象
      1. 使用 __block 去修饰这个对象，无需调用，因为 MRC 下不会对这个对象进行强引用

另外，还能通过将对象作为 block 的参数传入，因为 实参 的空间是在栈中，由 栈 对这个形参进行生命周期的管理

七、 __block

  • 对于 基本类型 （int long）变量

    会把上文封装的结构体的 forwarding 指针 指向这个变量的地址

  • 对于 OC 对象

    ARC：会将其 栈指针 copy 一份 （两个指针指向的（堆）地址一样），而且会对对象进行 retain

    MAC：会将其 指针 copy 一份 （两个指针指向的地址一样）
