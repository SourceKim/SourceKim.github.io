GCDTimer - 更精确的计时器

GCD Timer，顾名思义是一个计时器，由 GCD 封装，使用 Runloop 中的 timer source 实现，更加接近底层。

相较于 NSTimer，GCD Timer 更加 精确。

本篇会介绍 GCD Timer 的完整 api；以及使用的坑，比如 EXC_BAD_INSTRUCTION 崩溃，GCD Timer 不运作等。

代码位于 我的 iOS Experiment 仓库 的 GCDTimer 中。

一、Api 介绍

1、创建 timer source

dispatch_source_create

dispatch_source_t
dispatch_source_create(dispatch_source_type_t type,
	uintptr_t handle,
	unsigned long mask,
	dispatch_queue_t _Nullable queue);

参数：

  • type Source 的 类型，我们要创建一个 timer source，因此传入 DISPATCH_SOURCE_TYPE_TIMER
  • handle The underlying system handle to monitor。一般传入 0
  • mask A mask of flags specifying which events are desired。一般传入 0
  • queue Event 处理 block 回调的队列

2、设置 timer source

dispatch_source_set_timer

void
dispatch_source_set_timer(dispatch_source_t source,
	dispatch_time_t start,
	uint64_t interval,
	uint64_t leeway);

参数：

  • source 要设置的 source，使用第 1 步创建的即可
  • start 开始时间，如果要立马开始，设置 DISPATCH_TIME_NOW 即可
  • interval 循环间隔，比如 0.1s 回调一次设置为 0.1 * NSEC_PER_SEC
  • leeway 可以忍受的误差时间，如果设置成 3s，则回调的时候误差会 尽量 控制在 3s，一般设为 0

3、设置 timer 的事件回调

dispatch_source_set_event_handler

void
dispatch_source_set_event_handler(dispatch_source_t source,
	dispatch_block_t _Nullable handler);

参数：

  • source 要设置的 source，使用第 1 步创建的即可
  • handler 回调的 block

4. 恢复 timer

    dispatch_resume(timer);

5、挂起 timer

    dispatch_suspend(timer);

6、取消 source

    dispatch_source_cancel(timer);

7、销毁 timer

    timer = nil;

二、注意点

1、GCD Timer 不回调

GCD Timer 需要用成员变量去持有，否则出了作用域就释放了，不像 NSTimer 不需要持有也能运行。

假设我们这样设置：

    dispatch_source_t timer = dispatch_source_create(DISPATCH_SOURCE_TYPE_TIMER, 0, 0, timerQueue);
    dispatch_source_set_timer(timer, DISPATCH_TIME_NOW, 0.1 * NSEC_PER_SEC, 0);
    dispatch_source_set_event_handler(timer, ^{
        NSLog(@"hi");
        if (cnt == 5) {
            [self stop];
        }
        cnt++;
    });
    dispatch_resume(timer);

此时的 event 是不会回调的！

必须要持有它：

    @implementation ViewController {
        
        dispatch_source_t timer;
    }

再赋值才行：

    timer = dispatch_source_create(DISPATCH_SOURCE_TYPE_TIMER, 0, 0, timerQueue);
    dispatch_source_set_timer(timer, DISPATCH_TIME_NOW, 0.1 * NSEC_PER_SEC, 0);
    dispatch_source_set_event_handler(timer, ^{
        NSLog(@"hi");
        if (cnt == 5) {
            [self stop];
        }
        cnt++;
    });
    dispatch_resume(timer);

2、调用 dispatch_resume 才会运行，且会立即运行一次

根据官方文档对 dispatch_source_create 的描述：

所有的 source 都是以 suspend 的状态创建的

仅当其 状态是 resume 才会运作，且会 立即回调一次

3、timer 置 nil 的时候发生 EXC_BAD_INSTRUCTION 崩溃

当 timer 的状态是 suspend 的状态下，将其赋值为 nil，就会 EXC_BAD_INSTRUCTION Crash!

需要检查是否有 resume & suspend 数量不匹配的问题。

正确的销毁做法是：

  • 先 cancel
  • 再设置成 nil
- (void)stop {
    dispatch_source_cancel(timer);
    timer = nil;
}
