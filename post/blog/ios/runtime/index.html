iOS 中 Runtime 详细介绍

 Posted on March 4, 2019  |  3 minutes  |  557 words  |  Kim

Runtime，运行时的库，是一套使用 C 和汇编的 API，因为 runtime 使得 OC 可以成为一个动态语言。平时我们定义的对象，大多都是编译后就是 runtime 的 objc_class 结构体；我们调用方法，编译之后会变成 objc_msgSend:，也就是通过给对象发送消息来调用方法，方法实际的实现在运行时才能被确定。

一、内存布局

  • objc_object 对象、实例
  • objc_class 类

指向：

  • object 的 isa 指向其 class
  • class 的 isa 指针指向 metaClass
  • class 的 superclass 指针指向其父类 （会一直到 NSObject）
  • metaClass 的 isa 指向 rootMetaClass
  • rootMetaClass 的 isa 指向其自身
  • rootMetaClass 的 superclass 指向 NSObject
  • NSObject 的 isa 指向 rootMetaClass
  • NSObject 的 superclass 指向 nil

关系如下所示：

实例对象 objc_object

#include <objc/objc.h>

@interface NSObject <NSObject> {
    Class isa OBJC_ISA_AVAILABILITY;
}

只有一个 isa，指向其类

类对象 objc_class

// objc_class继承于objc_object,因此
// objc_class中也有isa结构体
struct objc_class : objc_object {
    isa_t isa;
    Class superclass;
    // 缓存的是指针和vtable,目的是加速方法的调用
    cache_t cache;  
    // class_data_bits_t 相当于是class_rw_t 指针加上rr/alloc标志
    class_data_bits_t bits;  
    // 其他函数
}

  • isa

    存储了类的指针以及一些其他的信息。对象的方法是存储在类中的，当调用对象方法时，对象就是通过 isa 结构体找到自己所属的类，然后在类结构中找到方法

  • superclass

    父类指针。指向该类的父类。

  • cache

    方法缓存

  • bits

    通过计算得 class_rw_t

class_rw_t （read_write）

// 类的方法、属性、协议等信息都保存在class_rw_t结构体中
struct class_rw_t {
    uint32_t flags;
    uint32_t version;

    const class_ro_t *ro;
    
    // 方法信息
    method_array_t methods;
    // 属性信息
    property_array_t properties;
    // 协议信息
    protocol_array_t protocols;

    Class firstSubclass;
    Class nextSiblingClass;

    char *demangledName;
}

注意，有一个 class_ro_t 类型的 ro 指针，意思是 class_rw_t 包括 class_ro_t 的指针

class_ro_t （read_only）

// class_ro_t结构体存储了类在编译期就已经确定的属性、方法以及遵循的协议
// 因为在编译期就已经确定了，所以是ro(readonly)的，不可修改
struct class_ro_t {
    uint32_t flags;
    uint32_t instanceStart;
    uint32_t instanceSize;
#ifdef __LP64__
    uint32_t reserved;
#endif
    const uint8_t * ivarLayout;
    const char * name;
    // 方法列表
    method_list_t * baseMethodList;
    // 协议列表
    protocol_list_t * baseProtocols;
    // 变量列表
    const ivar_list_t * ivars;
    const uint8_t * weakIvarLayout;
    // 属性列表
    property_list_t *baseProperties;
};

class_ro_t 结构体存储了类在 编译期间 确定的属性、方法、协议以及变量

分类 category_t

typedef struct category_t {
    const char *name;
    classref_t cls;
    struct method_list_t *instanceMethods;
    struct method_list_t *classMethods;
    struct protocol_list_t *protocols;
    struct property_list_t *instanceProperties;
} category_t;
  • 类的名字（name）
  • 类（cls）
  • category中所有给类添加的实例方法的列表（instanceMethods）
  • category中所有添加的类方法的列表（classMethods）
  • category实现的所有协议的列表（protocols）
  • category中添加的所有属性（instanceProperties）

在 dyld 加载分类的时候，会把新的方法 attachNewMethod: 附加到原有的方法列表的 前面，也就是说，当有同名的方法的话，category 是不会覆盖原有的方法的，只是把它加到了类的方法列表的前面，此时就会 有一对的同名方法在类的方法列表里面，在调用的时候是按顺序去查找的，因此会先调用到 category 的方法，然后就不继续调用了。

二、Method、SEL、IMP

  • SEL 方法名

    类似于 C 的字符串，不带参数，因此同个类不可以有同名的方法（重载）

  • IMP 方法的实现，函数指针

    指向最终实现程序的内存地址的指针

  • Method 包含了 SEL & IMP

    /// An opaque type that represents a method in a class definition.代表类定义中一个方法的不透明类型
typedef struct objc_method *Method;
struct objc_method {
    SEL method_name                                          OBJC2_UNAVAILABLE;
    char *method_types                                       OBJC2_UNAVAILABLE;
    IMP method_imp                                           OBJC2_UNAVAILABLE;
}

三、关联对象

可以用来给 Category 添加属性

api

//  设置 关联对象
void objc_setAssociatedObject(id object, const void *key, id value, objc_AssociationPolicy policy)
// 获取 关联的对象
id objc_getAssociatedObject(id object, const void *key)
// 移除 关联的对象
void objc_removeAssociatedObjects(id object)
  • id object：被关联的对象
  • const void *key：关联的key，要求唯一
  • id value：关联的对象
  • objc_AssociationPolicy policy：内存管理的策略

内存管理策略 policy

typedef OBJC_ENUM(uintptr_t, objc_AssociationPolicy) {
    OBJC_ASSOCIATION_ASSIGN = 0,           /**< Specifies a weak reference to the associated object. */
    OBJC_ASSOCIATION_RETAIN_NONATOMIC = 1, /**< Specifies a strong reference to the associated object. 
                                            *   The association is not made atomically. */
    OBJC_ASSOCIATION_COPY_NONATOMIC = 3,   /**< Specifies that the associated object is copied. 
                                            *   The association is not made atomically. */
    OBJC_ASSOCIATION_RETAIN = 01401,       /**< Specifies a strong reference to the associated object.
                                            *   The association is made atomically. */
    OBJC_ASSOCIATION_COPY = 01403          /**< Specifies that the associated object is copied.
                                            *   The association is made atomically. */
};

关联对象的底层实现

  • 通过 AssociationManager 管理着一张全局的 HashMap - AssociationHashMap（全局哈希表）

  • 全局哈希表 的 key 是某个对象的指针，value 又是一个 HashMap - ObjectAssociationMap （对象哈希表）

  • 对象哈希表 的 key 是我们传入的键，value 是一个结构体，包含着 value & policy 两个内容

总结：

  • 关联对象并不是存储在被关联对象本身内存中，而是存储在 全局的统一 的一个 AssociationsManager 中，

  • 如果设置关联对象为 nil，就相当于是移除关联对象

四、Category & Extension 的区别

命名的区别

  • Category 可以在括号中写上 分类名
  • Extension 的括号不能有内容

作用的区别

  • Category 可以使用 @property，会自动生成 getter & setter，但是不会自动实现，有 .h .m 文件
  • Extension 只能添加 .h，其具体实现要在类的实现的 .m 文件中

注意：

  • Category 虽然能写 @property，但是在 其结构体中并没有属性列表，所以我们要给 Category 添加属性的话，要使用关联对象

  • 使用 Extension 必须要有源码，所以我们不能给 NSObject 添加分类

应用场景的区别

  • Category 可以用来实现 私有接口

    // someClass.m 
@interface someClass () 
-(void)extend; 
@end 
  
@implementation someClass 
// 所有声明在头文件或父类中方法的实现 
// 或者一些私有函数 
-(void)extend { 
    // implement private method here; 
} 
  
@end

  • Extension 可以用来实现 公有只读，私有读写 的属性

    @interface MyClass : NSObject
@property (retain, readonly) float value;
@end
  
// 私有的extension, 隐藏在主实现文件中.
@interface MyClass ()
@property (retain, readwrite) float value;
@end
