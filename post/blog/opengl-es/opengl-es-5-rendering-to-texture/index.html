OpenGL ES in iOS - 5 绘制到 texture (实现滤镜链)

  • 二、核心思路
  • 三、操作要点：

    • 3. 执行渲染 render

OpenGL 的纹理对象（FBO）的 attachment 除了可以与渲染对象（RBO）绑定直接输出之外，还可以与一个 2DTexture 绑定，这样就可以使用该特性，来对 Texture 进行多次修改（滤镜链），最后才将其渲染到 RBO 来展示结果。

本章的代码对应 仓库 中的 4.滤镜链

最终效果：

有两个滤镜，分别是 饱和度 和 亮度，然后动态渲染过去。

一、整体流程

整理的流程在 ViewDidAppear: 中实现，这样可以保证 layer 的 size 是自动布局之后的正确位置：

// 在这儿 setup，能保证 glLayer 的 size 是正确的，否则更新它的 frame 可能会出错
- (void)viewDidAppear:(BOOL)animated {
    [super viewDidAppear: animated];
    
    [self setupContext];
    [self setupLayer];
    [self setupPrograms];
    [self setupTempTexture: _image.size];
    [self setupImageTexure: _image];
    [self createBuffers];
    [self setupVAOs];
    
    [self render];
}
  • 1、 配置 Context
  • 2、 配置 Layer 来展示 RBO
  • 3、 配置 Programs，链接到饱和度和亮度 shaders
  • 4、 配置临时存储的纹理 tmpTexture
  • 5、 将 UIImage 图片上传到 OpenGL ES 的纹理 texture 中
  • 6、 构造一个 RBO 和两个 FBOs，两个 FBO 分别对应饱和度、亮度的 FrameBuffer
  • 7、 构造两个 VAO，分别为饱和度、亮度管理顶点属性
  • 8、 执行渲染，包含如下 3 步骤
    • 8.1 执行饱和度的渲染 DrawElement 到 tmpTexture
    • 8.2 执行亮度的渲染 DrawElement 到 RenderBuffer
    • 8.3 展示 RBO presentRenderbuffer:

二、核心思路

核心思路就是让 设定一个临时存储纹理，然后把 除了最后一个 的 FBO 都渲染到这个临时存储纹理上，最后一个渲染到 RBO 上即可。

对应到 Demo 中：

  • 我定义了第一个渲染是饱和度渲染，第二个也就是最后一个是亮度渲染
  • 将原图纹理(_imageTexture)通过饱和度程序渲染到临时纹理(_tmpTexutre) 上
  • 将 临时纹理(_tmpTexutre) 通过亮度程序渲染到 RBO 上

三、操作要点：

1. 创建临时纹理

和加载图片纹理类似：

  • 调用 glGenTextures 申请句柄
  • 调用 glBindTexture 使用纹理
  • 最后，最重要，使用 glTexImage2D 来开辟一个纹理数据存储区

调用 glTexImage2D 有如下个注意点：

  • 除一些 transform 变换的情况之外，开辟的 size，要和原图一致，才能保证原图在链中传递的时候大小不变
  • 由于该纹理在一开始的时候并没有图片数据，因此最后一个参数传入 NULL 即可
  • 空的纹理数据也需要指定 glTexParameteri 纹理参数

具体代码：

- (void)setupTempTexture: (CGSize)size {
    glGenTextures(1, &_tmpTexture);
    glBindTexture(GL_TEXTURE_2D, _tmpTexture);
    
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, size.width, size.height, 0, GL_RGBA, GL_UNSIGNED_BYTE, NULL);
    
    glBindTexture(GL_TEXTURE_2D, 0); // unbind
}

2. 让 FBO 渲染到 Texture 上

回顾一下，将 FBO 渲染到 RBO 上，我们调用的是

    glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, GL_RENDERBUFFER, _RBO);

将 RBO 附着到 FBO 的 Attachment 上。

而 OpenGL 还支持让我们把纹理渲染到 Texture 上，使用的是 glFramebufferTexture2D

    glFramebufferTexture2D (GLenum target, GLenum attachment, GLenum textarget, GLuint texture, GLint level)
  • target：渲染管理者类型，目前是 GL_FRAMEBUFFER
  • attachment 附着点 一般用 GL_COLOR_ATTACHMENT0
  • textarget 渲染的纹理目标类型，一般用 GL_TEXTURE_2D
  • texture 渲染的 texture ID
  • level mipmap 的层次，一般是 0
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, GL_TEXTURE_2D, _tmpTexture, 0);

因此，我们先选择 FBO，再调用该函数，那么当该 FBO 渲染的时候就会渲染到 texture 参数对应的纹理 ID 上了。

    glBindFramebuffer(GL_FRAMEBUFFER, _FBOs[0]); // 使用 FBO[0]，下面的激活 & 绑定操作都会对应到这个 FrameBuffer
    glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, GL_TEXTURE_2D, _tmpTexture, 0);

3. 执行渲染 render

对滤镜链的渲染要对链上的各个 FBO 都执行一次渲染，最后再调用 presentRenderbuffer 来将 RBO 展示。

该 demo 中链上有两个 FBO，因此步骤就是：

  • 执行饱和度的渲染 DrawElement 到 tmpTexture
  • 执行亮度的渲染 DrawElement 到 RenderBuffer
  • 展示 RBO presentRenderbuffer:

FBO 的渲染步骤如下：

  • a、选定渲染程序 glUseProgram
  • b、选择 FBO glBindFramebuffer
  • c、清理 FBO & 设置 Viewport glClearColor glClear glViewport
  • d、上传 uniform glUniform1f(i)，包括纹理和一些参数
    • 对于第一个 FBO，纹理使用原图纹理
    • 对于其他（这里是第二个）FBO，纹理使用临时纹理 _tmpTexture
  • e、使用对应的 VAO glBindVertexArray
  • f、画到画布上 glDrawElements

两个滤镜渲染的对应代码：

- (void)render_gray: (CGSize)clearSize {
    glUseProgram(_glPrograms[0]);
    glBindFramebuffer(GL_FRAMEBUFFER, _FBOs[0]);
    [self clearFBO: clearSize];
    glUniform1i(_imageTextureUniformLoc, _imageTextureBufferIndex);
    glUniform1f(_grayIntensityUniformLoc, _grayIntensity);
    glBindVertexArray(_VAOs[0]); // 使用 VAO 的好处，就是一句 bind 来使用对应 VAO 即可
    glDrawElements(GL_TRIANGLE_STRIP, 2 * 3, GL_UNSIGNED_SHORT, 0);
}

- (void)render_brightness: (CGSize)clearSize {
    glUseProgram(_glPrograms[1]);
    glBindFramebuffer(GL_FRAMEBUFFER, _FBOs[1]);
    [self clearFBO: clearSize];
    glUniform1i(_tmpBufferTextureUniformLoc, _tmpTextureBufferIndex);
    glUniform1f(_brightnessUniformLoc, _brightness);
    glBindVertexArray(_VAOs[1]); // 使用 VAO 的好处，就是一句 bind 来使用对应 VAO 即可
    glDrawElements(GL_TRIANGLE_STRIP, 2 * 3, GL_UNSIGNED_SHORT, 0);
}

最后调用一下 presentRenderbuffer 即可绘制到 _glLayer 上：

- (void)present {
    glBindRenderbuffer(GL_RENDERBUFFER, _RBO);
    [_ctx presentRenderbuffer: GL_RENDERBUFFER];
}

四、总结

绘制到 texture 来实现滤镜链关键就是要使用创建一个空的 GL_TEXTURE_2D 来承载 FBO 的绘制内容（因为 FBO 只做管理，不保存纹理），然后使用 glFramebufferTexture2D api 告诉 OpenGL ES 该 FBO 绘制的结果会保存在创建的 GL_TEXTURE_2D 上。
