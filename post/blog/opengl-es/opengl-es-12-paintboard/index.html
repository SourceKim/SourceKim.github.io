使用 OpenGL ES，在 iOS 上实现一个涂鸦绘画板，通过监听 touch 手势，获取到要绘制的点（GL_POINTS），最终渲染到屏幕上。

包含：笔刷 & 擦除（橡皮擦）基础功能。画笔颜色 & 画笔大小 功能可以通过变量配置。

核心在于 在每一次绘制完点之后，不清空 FrameBuffer，保留上一次绘制的内容，通过混合添加这次的绘制（或擦除）的点。

一、如何保留 FrameBuffer 上一次绘制的内容？

1、绘制的时候不清理 FrameBuffer

前面的代码中我们会在每一次 render 的时候都 clear 一下 FBO，但是由于我们要保留之前的内容，因此在该 demo 中不能在每次渲染都清空 FBO。

2、 CALayer 的 drawableProperties 字典中，设置 kEAGLDrawablePropertyRetainedBacking 为 true

    _glLayer.drawableProperties = @{
        kEAGLDrawablePropertyRetainedBacking: @(true), // Important !!!
        kEAGLDrawablePropertyColorFormat: kEAGLColorFormatRGBA8,
    };

先看下官方对 kEAGLDrawablePropertyRetainedBacking 的注释：

大致意思就是如果为 true，就会保留绘制后的 buffer 的内容，这样会导致性能下降和占用额外内存开销的问题。

假设不设置这个属性，每一次绘制完之后都会把渲染 buffer 清空，导致的后果就是展示 buffer 和 渲染 buffer 各自维护一组点，然后交替显示。

二、橡皮擦功能

通过 OpenGL ES 的 bleanding (混合) 功能实现。

  • 启用混合：

    glEnable(GL_BLEND);

  • 选择混合方法：

    • 当绘制模式之下

      glBlendFunc(GL_ONE, GL_ONE_MINUS_SRC_ALPHA);

    • 当橡皮擦模式之下

      glBlendFunc(GL_ZERO, GL_ONE_MINUS_SRC_ALPHA);

将 source 的 factor 设为 GL_Zero （即 0） 即可实现 橡皮擦 功能。

二、笔刷的实现

笔刷包含 3 个内容：

  • 笔刷纹理

  • 笔刷大小

  • 笔刷颜色

先看下两个 shader：

vertex shader:

attribute vec4 position;

uniform float pointSize;

void main (void) {
    gl_Position = position;
    gl_PointSize = pointSize;
}

fragment shader:

precision highp float;

uniform vec3 color;
uniform sampler2D brushTexture;

void main (void) {
    vec4 textureColor = texture2D(brushTexture, gl_PointCoord);
    gl_FragColor = vec4(color, 1.0) * textureColor;
}

通过 gl_PointSize 即可设置点的大小，也就是笔刷的大小。

笔刷的纹理通过一个 texture 来存着，然后通过 gl_PointCoord 可以获取到 片元相对于绘制的点的归一化坐标。

OpenGL 画出来的点都是一个个矩形，要组成一个圆就需要这样剖成多个矩形，而 gl_PointCoord 能获取到这些小矩形在 “圆” 中的相对的归一化坐标。

最终的颜色是通过传入的 color uniform 和在笔刷纹理上采样的颜色混合相乘即可。

三、监听 touch 转换成 OpenGL ES 的顶点 point

为了不侵入业务逻辑，我在 demo 中封装了 PaintBoardTouchManager 管理类管理手势事件。

该类的输入是 touch (began / moved / ended)，输出是一系列要绘制的点（(NSArray<NSValue *> *)points）。

  • 当输入是 touchBegan 的时候，输出只有一个点

  • 当输入是 touchMoved & touchEnded 的时候，需要获取 与上一个点之间的填充点

    - (NSMutableArray<NSValue *> *)getPoints: (CGPoint)from
                                toPoint: (CGPoint)to
                        withPointSize: (CGFloat)pointSize {
      
    CGFloat count = MAX(
                        ceilf(
                            sqrtf(
                                    powf(to.x - from.x, 2) +
                                    powf(to.y - from.y, 2)
                                    )
                            / pointSize),
                        1
                        );
      
    count += 10; // 经验值，这样可以让点稠密
      
    NSMutableArray *mPoints = [NSMutableArray array];
      
    for (int i = 0; i < count; i++) {
          
        CGFloat percent = (CGFloat)i / (CGFloat)count;
        [mPoints addObject: @(
        CGPointMake(from.x + (to.x - from.x) * percent,
                    from.y + (to.y - from.y) * percent)
        )];
    }
      
    return mPoints;
}

    通过将 from 到 to 分段，然后插入两段之间的值实现。

另外，如果需要更加流畅的效果，推荐使用 这篇博客 - lymanli 的 「神奇」的方法，其原理使用的是 bezier curve 实现，而我这里只是简单地进行连线。
