一、 认识 OpenGL ES 的坐标空间

手机的屏幕是一个 2D 的空间，但是当我们需要渲染一个 3D 的物体的时候，就需要将该物体“压平”，即投影到 2D 的空间中来完成展示。

当我们需要渲染一个 3D 物体的时候，需要传入的是一组描述该物体的三维顶点来描述物体的 形状，其位置信息是需要我们在进行坐标系转换（通过 MVP 矩阵）的时候确定的，并且最终要转换到屏幕上。

因此，在 OpenGL ES 中，我们人为地定义了一些坐标空间：

  • 本地空间 Local Space，以 (0, 0, 0) 为中心，描述物体的位置信息
  • 世界空间 World Space，多个物体在同一个世界的位置，即多个物体之间的相对位置
  • 观察空间 View Space，对于上述的同一个世界，我们可以有不同的观察角度，该空间是同一个角度观察多个物体的空间
  • 裁剪空间 Clip Space，世界肯定比我们要渲染的目标大，因此我们可以通过将观察到的世界存在于画布范围内的内容投影到我们的画布上，并且将多余的裁剪丢弃掉。裁剪过后会返回在 NDC (Normalized Device Coordinates 标准坐标系) 的归一化的点，范围在 -1 ~ 1 内
  • 屏幕空间 Screen Space，将上述归一化的坐标点转换到荧幕上视图的真实尺寸（如 640 X 480，glViewport 指定）的位置

二、MVP 矩阵

下图很好地描绘了各个空间之间的转换

  • 本地，是直接上传过来的顶点数据
  • 本地 -> 世界，通过 Model 矩阵
  • 世界 -> 观察，通过 View 矩阵
  • 观察 -> 裁剪，通过 Projection 矩阵
  • 裁剪 -> 屏幕，这一步是在设置了 glViewPort 之后，OpenGL ES 自动进行的

因此我们主要来了解上面的 3 个矩阵。

通过 MVP 矩阵去变换原始顶点数据的过程主要发生在 vertex shader 中，我们可以声明 3 个 mat4 代表 MVP 矩阵，使用 uniform 来修饰。

    uniform mat4 modelMatrix;
    uniform mat4 viewMatrix;
    uniform mat4 projectionMatrix;

然后将传入的顶点与所有矩阵做 矩阵相乘，即可得到最后裁剪空间的坐标点。

    gl_Position = projectionMatrix * viewMatrix * modelMatrix * vec4(position, 1.0);

注意，OpenGL 使用的 列主序（Column Major） 矩阵，因此要 反着相乘，理解这个调用需要从后往前读，即：

顶点数据 * model 矩阵 * view 矩阵 * project 矩阵

另外，由于需要考虑到万向节锁的问题，因此要采用四元数，原本用 n 组每组 3 个可以描述物体的 3D 坐标，但是要使用四元数，因此要多加一位初始为 1 的值到 position，所以就变成了上面的 vec4(position, 1.0)

通常在构造矩阵矩阵的时候，我们会使用 GLKit 去辅助，但是也有必要了解其原理，以后期进行一些自定义的行为。

1. Model 矩阵

model 矩阵决定的是物体在 在世界坐标系 的位置和大小等属性，一般用以对物体进行 平移、缩放、旋转。

  • 平移矩阵

    对应 api：GLKMatrix4MakeTranslation(<#float tx#>, <#float ty#>, <#float tz#>)

  • 缩放矩阵

    对应 api：GLKMatrix4MakeScale(<#float sx#>, <#float sy#>, <#float sz#>)

  • 旋转矩阵

    对应 api：GLKMatrix3MakeRotation(<#float radians#>, <#float x#>, <#float y#>, <#float z#>)

    第一个参数是旋转的弧度，如果需要从角度转换，则可以调用 GLKMathDegreesToRadians(Degrees) 去转换

    后面 3 个参数分别是绕哪个轴进行旋转，比如要绕 x 轴旋转 30°，则将 x 设为 1，其他设为 0 即可。

    GLKMatrix3MakeRotation(GLKMathDegreesToRadians(30), 1, 0, 0);

    • 绕 x 轴旋转

    • 绕 y 轴旋转

    • 绕 z 轴旋转

    注意：

    • OpenGL 使用右手规则做旋转，所以当角度为正数的时候，是逆时针旋转；当角度为负数的时候，是顺时针旋转

    • 矩阵乘法不满足交换律，变换的组合顺序不同会导致结果不一样

2. View 矩阵

观察矩阵，有时也称为 摄像机(camera) 或者 人眼(eye)。

该矩阵是让我们假定有一台摄像机，能在不同的位置去拍摄世界坐标系的物体，最终获得在该 摄像机观察之下的坐标。这台摄像机拥有他自己的 方向 & 位置，方向通过一个三维的 postition 向量表示；位置则通过 3 个三维的向量表示，分别是 Direction / Right / Up，代表着摄像机指向的方向 / 右轴 / 上轴，这 3 个向量 两两垂直。最终形成了一个以 position 为原点，Direction / Right / Up 为 3 个方向的 三维坐标系！

  • position 摄像机在世界坐标系的位置

    OpenGL 的 z 轴正方形是从屏幕指向我们的，所以要让摄像机原理 target，需要往 z 的正方向移动

  • direction 摄像机指向 target 的反方向的向量

    注意是反方向，即 target 指向摄像机的方向。

    direction = (target 的 position) - (camera 的 position)

  • right 摄像机右面的方向向量

    右轴，可以这样计算：

    先定义 世界坐标的 y 方向向量 WorldUp，这个向量指向世界的上面，然后叉乘（cross / 向量积）上面的 direction 即可得到同时垂直于 WorldUp 和 direction 的向量，再进行归一化就可以得到我们的右轴。

    right = Normalize(WorldUp X direction)

  • up 摄像机上面的方向向量

    上面也提到了，叉乘两个向量可以得到同时垂直两个轴的向量，所以 摄像机的上轴 可以直接由 direction 和 right 叉乘可得

    up = direction X right

在 OpenGL 中，我们使用 LookAt 矩阵，来构造一个 “摄像机”

  • R - right 摄像机的右轴
  • U - up 摄像机的上轴
  • D - direction 摄像机的方向
  • P - position 摄像机在世界坐标的位置

通过上面的计算可以知道：

  • D 可以通过 观察的目标位置（TargetPosition） 和 摄像机的位置（position P） 做减法得到

  • R 可以通过 世界的上方向（WorldUp） 和 摄像机的方向（direction D）做叉乘得到

  • U 可以通过 摄像机方向（D） 和 右轴（R）做叉乘得到

因此，要构建这个矩阵，我们只需要 3 个参数即可：观察的目标位置（TargetPosition）& 摄像机的位置（position P） & 世界的上方向（WorldUp）即可

GLKit 上为我们提供了这个函数构建 LookAt 矩阵

    GLKMatrix4 GLKMatrix4MakeLookAt(float eyeX, float eyeY, float eyeZ,
                                    float centerX, float centerY, float centerZ,
                                    float upX, float upY, float upZ)

  • eyeX & eyeY & eyeZ 代表 摄像机的位置向量，注意 z 为正数，代表让摄像机远离屏幕靠近我们，为负数的时候代表我们从屏幕的后面观察，得到的结果自然就是一个左右翻转的图片

  • centerX & centerY & centerZ 代表要观察的目标的位置向量，当观察世界的中心的时候，一般传入 (0, 0, 0)

  • upX & upY & upZ 代表世界的上方向（WorldUp），一般是 (0, 1, 0)

比如要把摄像机放到 (0, 0, 2) 的位置，去观察世界的中心 (0, 0, 0)，世界的上方是 (0, 1, 0)，则可以这么调用：

_viewMatrix = GLKMatrix4MakeLookAt(0, 0, 2, 0, 0, 0, 0, 1, 0);

3. projection 投影矩阵

投影矩阵可以将上面摄像机观察到的顶点转换到裁剪坐标系，来决定在什么范围的顶点才显示（在大于近裁面 near 和小于远裁面 far 的才显示，太近或者太远的丢弃），以及如何显示（选择正交矩阵则不对顶点的远近做缩放，选择透视则会因为顶点的远近而发生 近大远小）

透视投影 Perspective Projection

在我们实际观察物体的时候，两个大小一样的物体，距离我们越近的看起来会越大，越远则看起来会很小，就像我们看星星的时候觉得他们很小，实际上他们的尺寸十分大，只是距离我们远而已。透视矩阵就是要模仿这么一个效果，实现 在裁剪面之间的物体看起来近大远小，裁剪面之外的物体不显示。

  • 裁剪的原理：通过定义一个视椎体，来将椎体之外的顶点丢弃。

  • 近大远小的原理：通过 vec4 最后一个分量 w 来实现，该值是由与观察者的距离决定，越远 w 就越大，越近 w 就越小，最后输出的时候 gl_Position = (x / w, y / w, z / w)，就导致越远 顶点的值就越小。

透视投影有两种定义方式：

第一个是 GLKMatrix4MakeFrustum

    GLKMatrix4MakeFrustum(float left, float right,
                          float bottom, float top,
                          float nearZ, float farZ)

这种是通过投影板的上下左右位置（相对于屏幕像素的值）以及近裁面和远裁面来定义

比如说：

// todo: 距离让纹理上下左右都留空

第二个是 GLKMatrix4MakePerspective

    GLKMatrix4MakePerspective(float fovyRadians, float aspect, float nearZ, float farZ)

  • fovyRadians 即视角大小
  • aspect 宽高比 width / height
  • nearZ 近裁面
  • fazZ 远裁面

一般都使用这种方式，比如可以这么定义：

    _projectionMatrix = GLKMatrix4MakePerspective(GLKMathDegreesToRadians(90),
                                                  self.view.bounds.size.width / self.view.bounds.size.height,
                                                  0.1,
                                                  10);

三、Demo 实现， 让图片转起来！

该 Demo 的目的是把一张 2D 的图片纹理渲染到屏幕上，然后让他在 3D 空间中旋转。

1. 实现原理

有两种方式实现：

a、 摄像机不移动，旋转物体

让摄像机位置固定，通过旋转物体的 model 矩阵

b、 通过移动摄像机，绕着图片跑圈

让摄像机位置绕着物体移动，物体固定

这里使用的是方法 2，让相机绕着纹理转圈，并且时时刻刻注视（LookAt）这个纹理

2. 具体代码

设置 MVP 矩阵，其中 view 矩阵的 z 距离距离物体是 2，x & y 都为 0

    _cameraDistance = 2;
    
    _modelMatrix = GLKMatrix4Identity;
    _viewMatrix = GLKMatrix4MakeLookAt(0, 0, _cameraDistance, 0, 0, 0, 0, 1, 0);
    _projectionMatrix = GLKMatrix4MakePerspective(GLKMathDegreesToRadians(90),
                                                  self.view.bounds.size.width / self.view.bounds.size.height,
                                                  0.1,
                                                  10);

每次渲染之前都更新一下 x 和 z 的值，保持 y 不变，这样就能实现在 同一个水平线上面（y 不变），相机和物体的世界坐标的距离不变（通过勾股定理计算出新的 x & z）

途中所示的 x’ & z’ 就是转换后的摄像机位于 x 轴和 z 轴的位置的值。

然后再更新一下当前的角度 _cameraDegree 即可。

- (void)moveCameraAroundTarget {
    
    float x = _cameraDistance * sin(GLKMathDegreesToRadians(_cameraDegree));
    float z = _cameraDistance * cos(GLKMathDegreesToRadians(_cameraDegree));
    
    _viewMatrix = GLKMatrix4MakeLookAt(x, 0, z, 0, 0, 0, 0, 1, 0);
    
    if (_cameraDegree == 360) {
        _cameraDegree = 0;
    } else {
        _cameraDegree += 3;
    }
    
}
