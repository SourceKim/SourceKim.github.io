一、立方体的组成

在画一个立方体之前，我们需要了解一下如何组成一个立方体

如图所示，一个立方体是由 8 个顶点、6 个面 组成的

在 OpenGL 中可以轻易地将他们定义出来：

1. 顶点定义

8 个顶点分别对应图中的标号，我们可以假设 OpenGL 的 3D 坐标轴的中心点在立方体的中心，X 轴向右指向，Y 轴向上，Z 轴从屏幕指向我们：

static const GLfloat vertices[] = {
    -1, 1, 1,// 0
    1, 1, 1, // 1
    1, -1, 1, // 2
    -1, -1, 1, // 3
    -1, 1, -1, // 4
    1, 1, -1, // 5
    1, -1, -1, // 6
    -1, -1, -1 // 7
};

注释的标号对应图中的序号。

2. 面的定义

由于 EBO 可以更加直观地表示面，因此我们这里可以直接使用 EBO 来定义 6 个面（在我们之前的文章也使用过，具体参考前面）。

需要注意的是，在我们使用 EBO 去画 “面” 的时候，每 1 个矩形的面要由 2 个三角形组成，因此一共需要定义 6 * 2 = 12 个三角形索引：

static const GLushort indices[] = {
    
    // 正面
    0, 1, 3,
    1, 2, 3,
    
    // 右面
    1, 2, 6,
    1, 5, 6,
    
    // 背面
    4, 5, 7,
    5, 6, 7,
    
    // 左面
    3, 4, 7,
    0, 3, 4,
    
    // 上面
    0, 1, 4,
    1, 4, 5,
    
    // 下面
    2, 3, 6,
    3, 6, 7,
    
};

画出来的立方体能 “穿透”，因为 我们没有配置深度测试（DepthTest），因此显示的时候会把后绘制的像素覆盖先绘制的，而不是根据深度显示。

三、深度测试

为了让 同一位置的像素根据深度决定哪次计算是应该在前面，我们需要为上下文配置 深度测试。

1. 打开深度测试

默认情况下，上下文的 深度测试是关闭的，我们可以通过 glEnable 来打开:

    glEnable(GL_DEPTH_TEST);

注意该方法是对上下文配置的，所以我们至少要在 setCurrentContext 之后调用。

上下文中默认的深度方法是 GL_LESS，代表深度小的显示，我们可以不显式地设置：

    glDepthFunc(GL_LESS);

其他的深度方法可以自行查阅。

2. 申请深度缓存存储区，并附着到 FBO 上

这一步 十分重要！！！ 否则将没有地方去存储深度信息，即便开启了深度测试也没有用！

分为如下步骤：

  • ①、 先定义一个深度的 buffer

        GLUint _depthRBO

  • ②、 申请 & 使用这个 buffer

    注意深度 Buffer 是一个 RenderBuffer

        glGenRenderbuffers(1, &_depthRBO);
    glBindRenderbuffer(GL_RENDERBUFFER, _depthRBO);

  • ③、 分配存储区

    和渲染用的颜色 RBO 一样，也需要分配存储区，但是不需要使用 Context 去申请，也不需要绑定到 Layer

        glRenderbufferStorage(GL_RENDERBUFFER,
                          GL_DEPTH_COMPONENT16,
                          _glLayer.bounds.size.width,
                          _glLayer.bounds.size.height);

    这里代表我们要申请一个 每个纹理单元包含 16 位的深度信息，大小和 _glLayer 一样的 RenderBuffer 的存储区

  • ④、 附着到 FBO 的深度 Attachment 上

        glBindFramebuffer(GL_FRAMEBUFFER, _FBO); // 使用 FBO[1]，下面的激活 & 绑定操作都会对应到这个 FrameBuffer

    glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_RENDERBUFFER, _depthRBO);

    需要注意的是，附着到 FBO 的深度 Attachment，第二个参数要传入 GL_DEPTH_ATTACHMENT

因此，整体流程中的 createBuffers 完整代码如下：

    - (void)createBuffers {
        
        // 创建 RBO Render Buffer Object
        glGenRenderbuffers(1, &_RBO);
        glBindRenderbuffer(GL_RENDERBUFFER, _RBO);
        [_ctx renderbufferStorage: GL_RENDERBUFFER fromDrawable: _glLayer];
        
        // 申请深度渲染缓存
        glGenRenderbuffers(1, &_depthRBO);
        glBindRenderbuffer(GL_RENDERBUFFER, _depthRBO);
        // 设置深度测试的存储信息
        glRenderbufferStorage(GL_RENDERBUFFER,
                            GL_DEPTH_COMPONENT16,
                            _glLayer.bounds.size.width,
                            _glLayer.bounds.size.height);
        
        // Then, FBOs..
        
        // 创建 FBO Frame Buffer Object
        glGenFramebuffers(1, &_FBO);
        
        // 配置 FBO - Render FrameBuffer：
        glBindFramebuffer(GL_FRAMEBUFFER, _FBO); // 使用 FBO，下面的激活 & 绑定操作都会对应到这个 FrameBuffer
        
        glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_RENDERBUFFER, _depthRBO); // 附着深度 RBO
        glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, GL_RENDERBUFFER, _RBO); // 附着渲染的颜色 RBO
    }

3. 绘制前清理深度 buffer

由于之前的 demo 不存在深度 buffer，因此在绘制前仅需要清理 color buffer 即可：

- (void)clearFBO: (CGSize)size {

    glClearColor(0, 0, 0, 1);
    glClear(GL_COLOR_BUFFER_BIT);
    
    glViewport(0, 0, size.width, size.height);
}

现在加入深度信息，如果不清理的话 就会造成黑屏：

- (void)clearFBO: (CGSize)size {

    glClearColor(0, 0, 0, 1);
    glClearDepthf(1);
    glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT);
    
    glViewport(0, 0, size.width, size.height);
}

glClearDepthf(1); 代表默认深度是 1

glClear(GL_COLOR_BUFFER_BIT | GL_DEPTH_BUFFER_BIT); 代表要清理颜色 buffer 和深度 buffer

五、两个注意点：

1、使用 EBO 绘制

这里使用了 EBO，因此在绘制的时候需调用 glDrawElements，和前面不一样的是，该 demo 中绘制的顶点是不贡献的，因此第一个参数要使用 GL_TRIANGLES 来代表 12 个 独立的三角形。

    glDrawElements(GL_TRIANGLES, sizeof(indices), GL_UNSIGNED_SHORT, 0);

2、presentRenderbuffer 使用哪个 buffer？

以往我们 present 都是直接调用 presentRenderbuffer 的：

- (void)present {
    [_ctx presentRenderbuffer: GL_RENDERBUFFER];
}

但是该 demo 中，createBuffers 这一步我们最后 bind 的 RenderBuffer 是我们的深度 buffer，如果直接 presentRenderBuffer 就会展示了深度 buffer，也就是一片白屏（因为 glClearDepthf(1); 传入的是 1）。

因此我们要在 preset 之前，选择好 color RenderBuffer：

- (void)present {
    glBindRenderbuffer(GL_RENDERBUFFER, _RBO);
    [_ctx presentRenderbuffer: GL_RENDERBUFFER];
}

当然，这一步也可以在 createBuffers 的时候进行，因为后续上下文的 RBO 不再改动了，在这里进行更加严谨安全。
