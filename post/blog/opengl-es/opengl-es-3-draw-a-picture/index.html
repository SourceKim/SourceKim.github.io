本篇的最终目的是通过 OpenGL ES 的 glTexImage2D API，将一张 2D 的 JPG 图片，转换成纹理，然后画到 layer 上。

最后的实现结果：

整体步骤和上一篇：OpenGL ES - 2 画一个三角形 基本一样：

我们回顾一下，如何画一个三角形：

    [self setupContext];
    [self setupLayer];
    [self createBuffers];
    [self setupProgram];
    [self setupDrawable];
    [self renderVertex_VAO_Index];
    [self present];
    1. 设置 Context
    1. 设置 layer
    1. 构建和绑定 FBO & RBO （RBO 绑定 layer）
    1. 构建和链接着色程序（vertex & fragment shader）
    1. 清屏 + Viewport
    1. 离屏渲染 （drawArray 或者 drawPrimitive）
    1. 展示离屏渲染的 buffer （Present Drawable）

一、那么，如何用 OpenGL ES 绘制一张图片到 layer 上？

我们需要：

    1. 先把图片转换成 纹理数据
    1. 然后将 纹理数据 上传 + 绑定 到我们的 fragment shader 上 （uniform）
    1. 在 fragment shader 上根据 纹理坐标 在 纹理数据 上采样并输出即可

二、具体步骤

类似上一篇：

    [self setupContext];
    [self setupLayer];
    [self createBuffers];
    [self setupProgram];
    [self setupDrawable];
    [self setupImageTexure: [UIImage imageNamed: @"avatar.JPG"]];
    [self renderVertex_VBO_Index];
    [self present];

大体一样，有三处不同：

    1. 在 fragment shader 中，需要接收纹理数据
    1. 不需要上传 Color 到 vertex shader，取而代之的是纹理的顶点坐标
    1. 在 离屏渲染 之前，需要上传图片纹理

vertex shader & fragment shader

  • vertex shader

    image.vsh

    attribute vec3 position;
attribute vec2 texCoor;

varying vec2 outTexCoor;

void main()
{
    gl_Position = vec4(position, 1.0);
    outTexCoor = texCoor;
}
    • 直接将传入的 顶点数据（即 纹理坐标 texCoor）传入到 fragment shader，这一步会对其进行自动插值计算。（下一个小节会说怎么上传纹理坐标）

  • fragment shader

    image.fsh

    varying highp vec2 outTexCoor;

uniform sampler2D tex;

void main()
{
    gl_FragColor = texture2D(tex, outTexCoor);
}

    • 使用一个 vec2 来接收从 vertex shader 传过来的 纹理坐标(outTexCoor) 参数，使用 varying 修饰，就可以自动进行插值；

    • 使用一个 sampler2D 类型的 uniform 变量来接收 （2D纹理）；

    • 在纹理 tex 上对 outTexCoor 位置的点进行采样，并且将采样的点作为该片元的颜色。

纹理坐标 texCoor 的上传

将 texCoor 作为顶点的属性，类似于 顶点数据 上传：

此处只列出使用 VBO 上传，不使用 VBO / VAO 和使用 VAO 上传的方式可以参考我的上一篇文章：OpenGL ES - 2 画一个三角形

    glBindBuffer(GL_ARRAY_BUFFER, VBO[1]);
    glBufferData(GL_ARRAY_BUFFER, sizeof(texCoor), texCoor, GL_STATIC_DRAW);
    glVertexAttribPointer(_texCoorLoc, 2, GL_FLOAT, GL_FALSE, 0, 0);
    glEnableVertexAttribArray(_texCoorLoc);

注意，OpenGL 的纹理坐标系是 0-1，左下角是 (0, 0)，右上角是 (1, 1)

纹理坐标：

static const GLfloat texCoor[] = {
    0, 0, // 左下角
    1, 0, // 右下角
    0, 1, // 左上角
    1, 1, // 右上角
};

上传图片纹理 最重要的一步

1. 首先，需要将 UIImage 转成字节数据 void *

    CGImageRef cgImageRef = [image CGImage];
    GLuint width = (GLuint)CGImageGetWidth(cgImageRef);
    GLuint height = (GLuint)CGImageGetHeight(cgImageRef);
    CGRect rect = CGRectMake(0, 0, width, height);
    
    CGColorSpaceRef colorSpace = CGColorSpaceCreateDeviceRGB();
    void *imageData = malloc(width * height * 4);
    CGContextRef context = CGBitmapContextCreate(imageData,
                                                 width,
                                                 height,
                                                 8,
                                                 width * 4,
                                                 colorSpace,
                                                 kCGImageAlphaPremultipliedLast | kCGBitmapByteOrder32Big);
    
    CGContextTranslateCTM(context, 0, height); // 所有内容下移 height
    CGContextScaleCTM(context, 1.0f, -1.0f); // 翻转
    CGColorSpaceRelease(colorSpace);
    CGContextClearRect(context, rect);
    CGContextDrawImage(context, rect, cgImageRef);

这里的 imageData 就是要上传到 OpenGL ES de 字节数据

注意 UIKit 的坐标系原点是在左上角，而 OpenGL ES 的纹理坐标系的原点是在左下角，因此绘图的时候需要将图片 上下颠倒：

    CGContextTranslateCTM(context, 0, height); // 所有内容下移 height
    CGContextScaleCTM(context, 1.0f, -1.0f); // 翻转

直接翻转 context 即可。

2. 上传 2D 纹理，返回纹理句柄 ID

    glEnable(GL_TEXTURE_2D);
    GLuint textureID;
    glGenTextures(1, &textureID);
    glBindTexture(GL_TEXTURE_2D, textureID);
    
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_CLAMP_TO_EDGE);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
    glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);
    
    glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, width, height, 0, GL_RGBA, GL_UNSIGNED_BYTE, imageData);
    
    glBindTexture(GL_TEXTURE_2D, 0);

此处的 glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA, width, height, 0, GL_RGBA, GL_UNSIGNED_BYTE, imageData); 就是将 2D 纹理的字节数据 imageData 上传到前面 bind 的 textureID 处。

和其他句柄一样，纹理的句柄也需要先 gen 再 bind，中间的一些参数配置在下一篇文章会详解。

3. 激活 & 指定 纹理数据 到 fragment shader 的 uniform 处

现在在 OpenGL ES 上下文已经有了纹理的数据，我们也有访问、操作这个纹理的句柄了，现在要做的就是能让我们 fragment shader 也能访问到它，才能达到绘制它的目的。

    GLuint textureId = [self loadImageTexture: image];
    
    glActiveTexture(GL_TEXTURE3); // 使用（激活） 3 号纹理 buffer， 也可以是 GL_TEXTURE0 + 3
    glBindTexture(GL_TEXTURE_2D, textureId); // 将上面激活的纹理，与 加载纹理后返回的 id 绑定
    glUniform1i(_texUniformLoc, 3); // 将 激活的 3 号纹理，指定到 uniform 中，给着色器使用

uniform 只能是一些基本数据类型（int / float / mat …），但是无法是一段纹理数据，因此此处我们需要使用一个 int 索引，来把纹理数据映射到索引（此处是 3）处。

总共 4 行代码：

  • 1、 第一行就是 2. 上传 2D 纹理，返回纹理句柄 ID 中获得的纹理句柄 textureId

  • 2、 第二行激活一个纹理 buffer，我们使用 3 号 buffer

    OpenGL ES 提供我们至少 16 个纹理的 buffer，可以理解为 GPU 纹理数据的存储区

    glActiveTexture 激活纹理，可以理解为在显存申请一块纹理存储区，默认 0 号是活跃的，如果使用 0 号，则不需要调用 glActiveTexture。如果 激活、活跃，不好理解的话就理解为 使用这个纹理，在其他纹理被使用之前，所有的纹理操作都将作用到当前 active 的纹理。此处是为了使用 3 号 buffer，给下一步 纹理绑定用。

  • 3、 第三行就是告诉 OpenGL ES 这个状态机，现在我们要绑定 textureId 这个 2D 纹理（GL_TEXTURE_2D）到 当前活跃的 buffer，即上一步激活的 buffer。

  • 4、 第四行使用 glUniform1i 来将 3 号索引的纹理 buffer （也就是前两部 激活 & 绑定了纹理 id）的 buffer 上传（指定）到 fragment shader 的 _texUniformLoc 的位置

    _texUniformLoc 是 uniform 在 shader 中的位置，可以通过 _texUniformLoc = glGetUniformLocation(_glProgram, "tex"); 获取。

实际上，我们的目的是要将 textureId 和 _texUniformLoc 进行绑定，如果有一个 api 可以直接对应会更加方便，但是 OpenGL 在他们的中间引入了一个中间者，即前文的 索引，这样可以使得我们更加灵活地操作纹理，所以我们的操作就变成了：

textureId <-绑定-> GL_TEXTURE3

GL_TEXTURE3 <-绑定-> _texUniformLoc

间接地连接了 textureId 和 _texUniformLoc
