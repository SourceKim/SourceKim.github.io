OpenGL ES 获取 Uniform 的位置 glGetUniformLocation 返回 -1

 发表于 May 18, 2020  |  1 分钟  |  134 个字  |  Kim
uniform sampler2D lumaTexture;
uniform sampler2D chromaTexture;

uniform highp mat3 YUV_To_RGB_Matrix;
uniform highp vec3 YUV_Translation;

varying highp vec2 outTexCoor;

void main() {
    
    highp vec3 yuv, rgb;
    
    yuv.r = texture2D(lumaTexture, outTexCoor).r;
    yuv.gb = texture2D(lumaTexture, outTexCoor).rg;
    
    rgb = YUV_To_RGB_Matrix * (yuv + YUV_Translation);
    
    gl_FragColor = vec4(rgb, 1.0);
}

通过 glGetUniformLocation(_glProgram, "chromaTexture"); 返回一直是 -1

一番搜索才发现，原来是自己复制粘贴的笔误，将 chromaTexture 写成了 lumaTexture，即 chromaTexture 根本没有在 shader 中使用！

当该 uniform 没有被使用的时候，其 loction 就是 -1

另外：

没有被使用也包括，如果这个 uniform 没有对 gl_FragColor 产生影响，也认为 没有被使用。

比如说：

uniform sampler2D lumaTexture;
uniform sampler2D chromaTexture;

uniform highp mat3 YUV_To_RGB_Matrix;
uniform highp vec3 YUV_Translation;

varying highp vec2 outTexCoor;

void main() {
    
    highp vec3 yuv, rgb;

    highp vec3 test = texture2D(chromaTexture, outTexCoor).r;
    
    yuv.r = texture2D(lumaTexture, outTexCoor).r;
    yuv.gb = texture2D(lumaTexture, outTexCoor).rg;
    
    rgb = YUV_To_RGB_Matrix * (yuv + YUV_Translation);
    
    gl_FragColor = vec4(rgb, 1.0);
}

即使增加了 highp vec3 test = texture2D(chromaTexture, outTexCoor).r; 也同样会返回 -1，因为 gl_FragColor 和它没有一点关系，所以同样被认为没有被使用。
