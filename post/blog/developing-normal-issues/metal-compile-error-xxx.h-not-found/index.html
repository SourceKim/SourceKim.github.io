Metal 编译错误 CompileMetalFile XXX, fatal error: 'XXX.h' file not found

  • 问题原因
  • 解决方法
    • 方案 1、被包含文件和 .metal 放到同级目录
    • 方案 2、 使用完整引用路径（推荐）

在编译 Metal 文件的时候出现头文件找不到的问题。

报错信息：

CompileMetalFile XXX, fatal error: 'XXX.h' file not found

问题描述

当编译的时候，发现报 metal 的编译错误，当时我的目录结构是：

A 文件夹中有： B 文件夹，和文件 file0.h B文件夹中有：file1.metal

在 file1.metal 中导入 file0.h 头文件：

#include "file0.h"

即：

├── A
├── B
│   └── file1.metal
└── file0.h

问题原因

metal 中 引用其他文件需要其完整的相对路径 。

解决方法

方案 1、被包含文件和 .metal 放到同级目录

可以将 file1.metal 和 file0.h 置于 同级目录

方案 2、 使用完整引用路径（推荐）

方案 1 始终不优雅，更好的办法是 使用完整引用路径

#include "../file0.h"
