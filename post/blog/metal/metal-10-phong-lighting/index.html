本文将学习在 Metal 中渲染光照，主要采用的是 冯氏光照模型(Phong Lighting Model) 来实现。

将会涉及到如下的技术点：

  • 1、不使用索引（Index）绘制立方体（使用 36 组顶点）

  • 2、一次渲染中，绘制两个图像（一个 Encoder 多次 Draw Calls），一个作为物体，一个作为光源

  • 3、冯氏光照模型在 Metal 中的实现

最终实现的效果是物体自旋转，光源固定，物体表面因光线发生颜色变化。

一、冯氏光照模型

冯氏光照模型使用模型来表达复杂的光线和我们看到物体颜色之间的关系，主要有如下 3 个分量：

  • 环境光照(Ambient Lighting)：即使在黑暗的情况下，世界上也仍然有一些光亮(月亮、一个来自远处的光)，所以物体永远不会是完全黑暗的。我们使用环境光照来模拟这种情况，也就是无论如何永远都给物体一些颜色。
  • 漫反射光照(Diffuse Lighting)：模拟一个发光物对物体的方向性影响(Directional Impact)。它是冯氏光照模型最显著的组成部分。面向光源的一面比其他面会更亮。
  • 镜面光照(Specular Lighting)：模拟有光泽物体上面出现的亮点。镜面光照的颜色，相比于物体的颜色更倾向于光的颜色。

具体原理，请参考 OpenGL ES 篇 - 光照，此处不再赘述。

Metal 冯氏光照模型 shader

typedef struct
{
    float4 position [[position]];
    
    float3 vertexNormal;
    
    float3 fragmentPosition;
    
} VertexOut;

vertex VertexOut
PhongLightVertexShader(
                       uint vertexID [[ vertex_id ]],
                       constant packed_float3 *position [[ buffer(0) ]], // 位置
                       constant packed_float3 *normal [[ buffer(1) ]], // 法线
                       constant float4x4 *modelMatrix [[ buffer(2) ]], // 本地变换矩阵
                       constant float4x4 *viewMatrix [[ buffer(3) ]], // 观察矩阵
                       constant float4x4 *projectionMatrix [[ buffer(4) ]] // 投影矩阵
                       ) {
    VertexOut out;
    
    out.position = projectionMatrix[0] * viewMatrix[0] * modelMatrix[0] * float4(position[vertexID], 1);
    out.vertexNormal = (modelMatrix[0] * float4(normal[vertexID], 0)).rgb;
    out.fragmentPosition = (modelMatrix[0] * float4(position[vertexID], 1)).rgb;
    
    return out;
}

fragment float4
PhongLightFragmentShader(
                         VertexOut in [[ stage_in ]],
                         constant float3 *originColor [[ buffer(0) ]], // 当前渲染对象的颜色
                         constant bool *needLight [[ buffer(1) ]], // 是否需要渲染光线（光源无需反光）
                         constant float3 *lightColor [[ buffer(2) ]], // 光源颜色
                         constant float *ambientStrength [[ buffer(3) ]], // 环境光强度
                         constant float3 *lightPos [[ buffer(4) ]], // 光源的位置
                         constant float3 *eyePos [[ buffer(5) ]], // 眼睛（摄像机）的位置
                         constant float *specularStrength [[ buffer(6) ]] // 镜面反射的强度
                         ) {
    
    float3 color;
    
    if (needLight[0]) {
        
        // Ambient
        float3 ambient = ambientStrength[0] * lightColor[0];
        
        // Diffuse
        float3 norm = normalize(in.vertexNormal);
        float3 lightDir = normalize(lightPos[0] - in.fragmentPosition);
        
        float diff = max(dot(norm, lightDir), 0.0);
        float3 diffuse = diff * lightColor[0];
        
        // Specular
        float3 eyeDir = normalize(eyePos[0] - in.fragmentPosition); // 视线方向
        float3 reflectDir = reflect(-lightDir, norm); // 光线反射的方向
        
        float3 spec = pow(max(dot(eyeDir, reflectDir), 0.0), 32.0);
        float3 specular = specularStrength[0] * spec * lightColor[0];
        
        color = (ambient + diffuse + specular) * originColor[0];
        
    } else {
        
        color = originColor[0];
        
    }
    return float4(color, 1);
}

  • Vertex Shader

    参数：

    • position 顶点向量
    • normal 法线向量
    • modelMatrix 本地变换矩阵（物体和光源不同，因为他们有不同的位置坐标）
    • viewMatrix 观察矩阵（物体和光源相同，因为观察者，或者说摄像机用的是同一个）
    • projectionMatrix 投影矩阵（物体和光源相同）

    输出（VertexOut 结构体）：

    • position 顶点矩阵变换后的向量
    • vertexNormal 法线矩阵变换后的向量
    • fragmentPosition 片元的本地坐标变换后的位置向量

    注意，顶点着色器的输出会自动进行插值处理。

  • Fragment Shader

    参数：

    • in Vertex Shader 光栅化后插值的输出
    • originColor 当前渲染对象的颜色。本 demo 物体是红色，光源是白色
    • needLight 是否需要渲染光线。对于光源来说，无需渲染反光，该值为 false
    • lightColor 光源颜色。本 demo 是白色
    • ambientStrength 计算环境光强度。在原理中也介绍了，环境光一般是个很小的值，本例子是 0.2
    • lightPos 光源的位置。实际上就是光源 Model Matrix 的 translate 矩阵的组成向量
    • eyePos 眼睛（摄像机）的位置。实际上是 View Matrix 的 eye 组成分量
    • specularStrength 计算镜面反射的强度。本 demo 使用 0.5

    输出：

    • 对于光源而言，即 needLight 为 false，直接输出光源颜色，即 originColor

      color = originColor[0];

    • 对于物体而言，即 needLight 为 true，输出物体颜色经过光源影响后的颜色，即 originColor 和冯氏光照模型应用的效果

      color = (ambient + diffuse + specular) * originColor[0];

二、不使用索引（Index）绘制立方体（使用 36 组顶点）

对于一个普通立方体而言，可以使用 8 个顶点 6 个面共享的方式去绘制；但是由于此处各个面的同一个顶点 法线是不一样的，因此需要各个面定义不同的顶点。

立方体每个面由 2 个三角形构成，一共 6 个面，所以一共有 12 个面，而每个面由 3 个顶点组成，因此一共了 36 个顶点，对应的法线向量的数量也是 36 个。

在之前，我们说过 Metal 由于 内存对齐 的原因，要尽量用 float4 来表达顶点，但是这里如果要修改工作量就会很大（我是从 OpenGL 篇直接 copy 的顶点和法线），那么如何在 shader 中接收这种 每三个 float 为一组顶点 的定义呢？

在我们的 shader 中，我们可以使用 packed_float3 来表示这些顶点，代表的是每 3 个 float 为一组顶点，他们是 紧密排布 的。

最后在 Encoder 中绘制图元的时候，要使用 MTLPrimitiveTypeTriangle 来绘制 独立的三角形，其中 vertexCount 也就是顶点的数量是 36 个：


    // ......

    [encoder drawPrimitives: MTLPrimitiveTypeTriangle
                vertexStart: 0
                vertexCount: 36];

三、一次渲染两个物体

在图形渲染中，一次 根据顶点绘制图元到 FrameBuffer 的操作被称为一个 Draw Call，一次渲染两个物体要求我们每次 present 之前要有两次 Draw Call。在本 demo 中要求我们两次绘制分别绘制物体和光源到 FrameBuffer 中，然后再执行 present 操作。

因此我们的 render 函数要改成这样：

- (void)render {

    id<CAMetalDrawable> currentDrawable = [_layer nextDrawable];
    _renderTargetDesc.colorAttachments[0].texture = currentDrawable.texture;
    
    id<MTLCommandBuffer> commandBuffer = [_queue commandBuffer];
    commandBuffer.label = @"Command Buffer";

    // ......

    id<MTLRenderCommandEncoder> encoder = [commandBuffer renderCommandEncoderWithDescriptor: _renderTargetDesc];
    encoder.label = @"Object Render Command Encoder";
    
    // ......
    
    [self render_obj: encoder]; // 渲染旋转的物体
    
    [self render_camera: encoder]; // 渲染光源
    
    [encoder endEncoding];
    
    [commandBuffer presentDrawable: currentDrawable];
    
    [commandBuffer commit];
}

每次渲染的时候使用 1 个 Command Buffer，每个 Command Buffer 对应 1 个 Encoder，在每个 Encoder 中会有 2 次 Draw Calls，即 drawPrimitive: 的调用，分别在 render_obj: 和 render_camera: 两个方法中，他们都需要对 encoder 进行参数配置（setVertexBytes: / setFragmentBytes….）以及 1 次 drawPrimitive:，差异仅在参数的配置上。

渲染旋转的物体：

- (void)render_obj: (id<MTLRenderCommandEncoder>)encoder{
    
    bool needLight = true;
    
    [encoder pushDebugGroup: @"Obj Draw Call"];
    
    [encoder setVertexBytes: vertices
                     length: sizeof(vertices)
                    atIndex: 0];
    
    [encoder setVertexBytes: normals
                     length: sizeof(normals)
                    atIndex: 1];
    
    [encoder setVertexBytes: &_objModelMatrix
                     length: [MetalMatrix mm_matrixSize]
                    atIndex: 2];
    
    [encoder setVertexBytes: &_viewMatrix
                     length: [MetalMatrix mm_matrixSize]
                    atIndex: 3];
    
    [encoder setVertexBytes: &_projectionMatrix
                     length: [MetalMatrix mm_matrixSize]
                    atIndex: 4];
    
    [encoder setFragmentBytes: &_objColor length: sizeof(simd_float3) atIndex: 0];
    [encoder setFragmentBytes: &needLight length: sizeof(bool) atIndex: 1];
    [encoder setFragmentBytes: &_cameraColor length: sizeof(simd_float3) atIndex: 2];
    [encoder setFragmentBytes: &_ambientStrength length: sizeof(float) atIndex: 3];
    [encoder setFragmentBytes: &_lightPos length: sizeof(simd_float3) atIndex: 4];
    [encoder setFragmentBytes: &_eyePos length: sizeof(simd_float3) atIndex: 5];
    [encoder setFragmentBytes: &_specularStrength length: sizeof(float) atIndex: 6];
    
    [encoder drawPrimitives: MTLPrimitiveTypeTriangle
                vertexStart: 0
                vertexCount: 36];
    
    [encoder popDebugGroup];
}

渲染光源：

- (void)render_camera: (id<MTLRenderCommandEncoder>)encoder {
    
    bool needLight = false;
    
    [encoder pushDebugGroup: @"Camera Draw Call"];
    
    [encoder setVertexBytes: vertices
                     length: sizeof(vertices)
                    atIndex: 0];
    
    [encoder setVertexBytes: normals
                     length: sizeof(normals)
                    atIndex: 1];
    
    [encoder setVertexBytes: &_cameraModelMatrix
                     length: [MetalMatrix mm_matrixSize]
                    atIndex: 2];
    
    [encoder setVertexBytes: &_viewMatrix
                     length: [MetalMatrix mm_matrixSize]
                    atIndex: 3];
    
    [encoder setVertexBytes: &_projectionMatrix
                     length: [MetalMatrix mm_matrixSize]
                    atIndex: 4];
    
    [encoder setFragmentBytes: &_cameraColor length: sizeof(simd_float3) atIndex: 0];
    [encoder setFragmentBytes: &needLight length: sizeof(bool) atIndex: 1];
    
    [encoder drawPrimitives: MTLPrimitiveTypeTriangle
                vertexStart: 0
                vertexCount: 36];
    
    [encoder popDebugGroup];
}

在渲染物体的时候需要完整的光照参数，以及 needLight 是 true；

但是在渲染光源的时候仅需要光源的颜色，以及 needLight 为 false 便可以渲染。

PushDebugGroup / PopDebugGroup

Coomand Buffer 和 Command Encoder 都有这两个方法


- (void)pushDebugGroup:(NSString *)string

- (void)popDebugGroup

  • pushDebugGroup 传入一个字符串，用以在 debug 的时候将 一系列 GPU 操作分组，在栈中标记为开始

  • popDebugGroup 将栈顶的分组标记为结束，并且出栈

一进一出就成功地 标记了一组 GPU 的操作。

可以看到每个 DebugGroup 被包裹成了黄色的文件夹。
