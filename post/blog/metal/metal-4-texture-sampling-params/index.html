在 Metal 中进行纹理采样的时候，我们可以使用参数去控制一些特殊情况，比如：

  • 1、当纹理坐标超过纹理坐标的位置如何处理？

    • MTLSamplerAddressModeClampToEdge
    • MTLSamplerAddressModeRepeat
    • MTLSamplerAddressModeMirrorRepeat
    • MTLSamplerAddressModeClampToZero

  • 2、使用 mipmap 提升缓存命中率的时候采用什么算法？

    • MTLSamplerMipFilterNotMipmapped
    • MTLSamplerMipFilterNearest
    • MTLSamplerMipFilterLinear

通过将这些参数值设置到我们的 采样器 （sampler 或叫 MTLSamplerState） 中，然后在 shader 中对纹理采样的时候应该即可。

具体代码可以访问 仓库 的 3. 纹理采样参数详解

一、MipMap 技术

该技术通过预先对纹理进行缩小，可以在渲染小图的时候 提升缓存采样的命中率。比如一张图片，在比较 “远” 的地方展示就会比较小，加入使用高分辨率版本的图，会有性能损耗，效果也会变差。此时我们可以预先生成一堆小图，在合适的时候应该不同 size 的图去提供给采样使用。

当然，在 2D 图片的情况下，一般纹理的大小和屏幕的分辨率差不多，一般 mipmap 设置为 false。

当纹理的 size 和实际渲染的 size 有出入的时候，有如下方法可以过滤器可以使用：

  • MTLSamplerMipFilterNotMipmapped 不需要 mipmap
  • MTLSamplerMipFilterNearest 使用近邻法过滤，从周围的像素中取值作为当前的值
  • MTLSamplerMipFilterLinear 使用（双）线性过滤，当前的像素值由周围的像素加权后计算得来

二、处理采样点超过纹理大小的情况

在 Metal 中有如下定义

typedef NS_ENUM(NSUInteger, MTLSamplerAddressMode) {
    MTLSamplerAddressModeClampToEdge = 0,
    MTLSamplerAddressModeMirrorClampToEdge API_AVAILABLE(macos(10.11)) API_UNAVAILABLE(ios) = 1,
    MTLSamplerAddressModeRepeat = 2,
    MTLSamplerAddressModeMirrorRepeat = 3,
    MTLSamplerAddressModeClampToZero = 4,
    MTLSamplerAddressModeClampToBorderColor API_AVAILABLE(macos(10.12)) API_UNAVAILABLE(ios) = 5,
} API_AVAILABLE(macos(10.11), ios(8.0));

可以看到，iOS 中仅支持 4 种：

  • MTLSamplerAddressModeClampToEdge 将超出的部分使用边缘的像素点补齐
  • MTLSamplerAddressModeRepeat 将超出的部分重复使用原纹理补全
  • MTLSamplerAddressModeMirrorRepeat 将超出的部分重复使用镜像的原纹理补全
  • MTLSamplerAddressModeClampToZero 将超出部分使用 0 （黑色）进行补全，如果纹理有透明通道，则用 (0,0,0,0)，否则使用不透明的黑色 (0,0,0,1)

三、Metal 中使用采样器

Metal 的采样器有如下重要的选项可以设置：

  • normalizedCoordinates 纹理坐标形式是否归一化，默认是 true，代表纹理归一化到 [0, 1]，否则则是屏幕的像素坐标

XAddressMode：定义了在 x、y、z 坐标若超出了纹理的采样范围，使用哪种模式去采样，即 MTLSamplerAddressModeClampToEdge 、MTLSamplerAddressModeRepeat、MTLSamplerAddressModeMirrorRepeat、MTLSamplerAddressModeClampToZero 等。默认都是 MTLSamplerAddressModeClampToEdge

  • rAddressMode z 坐标（深度） 方向超出了纹理范围的采样模式
  • sAddressMode x 坐标（宽度） 方向超出了纹理范围的采样模式
  • tAddressMode y 坐标（高度） 方向超出了纹理范围的采样模式

XXXFilter：定义了当采样的大小和纹理大小有出入如何合并像素的选项，默认都是 MTLSamplerMinMagFilterNearest

  • minFilter 当采样像素小于纹理大小的选项
  • magFilter 当采样像素小于纹理大小的选项
  • mipFilter 当采样像素不等于纹理大小的选项

这些参数，我们可以构造 OC 的对象传入到 shader，也能直接在 shader 中定义。

1、在 OC 构造采样器 MTLSamplerState

我们可以使用 MTLDevice 的

/*!
 @method newSamplerStateWithDescriptor:
 @abstract Create a new sampler.
*/
- (nullable id <MTLSamplerState>)newSamplerStateWithDescriptor:(MTLSamplerDescriptor *)descriptor;

这个 api 去构造一个 id <MTLSamplerState>，需要传入一个 采样器描述对象 MTLSamplerDescriptor，而所有选项参数都是设置到该对象中的。

- (void)setupSamplerState {
    MTLSamplerDescriptor *samplerDesc = [MTLSamplerDescriptor new];
    samplerDesc.rAddressMode = MTLSamplerAddressModeClampToEdge;
    samplerDesc.sAddressMode = MTLSamplerAddressModeClampToEdge;
    samplerDesc.tAddressMode = MTLSamplerAddressModeClampToEdge;
    samplerDesc.minFilter = MTLSamplerMinMagFilterNearest;
    samplerDesc.magFilter = MTLSamplerMinMagFilterNearest;
    samplerDesc.mipFilter = MTLSamplerMipFilterNearest;
    
    _samplerState = [_device newSamplerStateWithDescriptor: samplerDesc];
}

在 shader 中作为函数的参数接收：

fragment float4
TextureSamplingFragmentShader(
                        VertexOut in [[ stage_in ]],
                        texture2d<float, access::sample> tex [[ texture(0) ]],
                              sampler texSampler [[ sampler(0) ]]
                        ) {
    float4 color = tex.sample(texSampler, in.texCoor);
    return color;
}

这个 sampler texSampler [[ sampler(0) ]] 意思就是该片元着色器接收一个采样器参数，设置在第 0 个位置。

然后再 Encoder 中，将这个 MTLSamplerState 通过指令上传到 shader，上传的 API 是 ：

- (void)setFragmentSamplerState:(nullable id <MTLSamplerState>)sampler atIndex:(NSUInteger)index;：

    [encoder setFragmentSamplerState: _samplerState atIndex: 0];

2、直接在 shader 中定义采样器

在 shader 中可以这样定义：

constexpr sampler texSampler(
                             coord::normalized,
                             address::repeat,
                             filter::linear
                             );

constexpr 是 C++ 修饰符， 代表 sampler 仅仅在编译期间声明定义一次，因此我们可以将该定义写在 shader 中，或者 shader 外都不会影响性能。

选项参数写在 变量名之后，意思是使用这些参数去初始化这个采样器。

具体的参数定义可以参考 Apple 文档 Metal 文档

标记有 default 的是默认值，不传入就使用该值。

3、两种方式总结

  • 在 App 中定义可以 更加灵活，如 demo 中就可以通过 UISegmentControl 去更新这些参数

  • 在 shader 中定义 更加方便，如果是采样的参数不变的话建议使用这种方式。
